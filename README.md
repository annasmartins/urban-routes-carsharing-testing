# Urban Routes - Carsharing Testing

Projeto de QA manual para a funcionalidade de carsharing do Urban Routes, com checklist de layout, checklist funcional, casos de teste e resumo de bugs.

## Sobre o projeto
Este repositório apresenta um projeto prático de QA manual voltado à funcionalidade de compartilhamento de carros do Urban Routes.

A atividade envolveu a preparação da documentação de teste e a execução dos testes planejados para validar layout, comportamento funcional, botão de reserva e lógica de locação do veículo.

## Objetivo
Validar a funcionalidade de carsharing do Urban Routes por meio de checklists, casos de teste e registro de bugs identificados durante a execução.

## Escopo analisado
- Layout do formulário de reserva
- Janela "Método de pagamento"
- Janela "Adicionar cartão"
- Lógica do botão "Reservar"
- Fluxo de locação de carro

## Ambientes utilizados
- Google Chrome — resolução 800x600
- Firefox — resolução 1920x1080

## Arquivos deste repositório
- `layout-checklist.md` → checklist de layout do formulário de reserva
- `payment-and-card-checklist.md` → checklist funcional de pagamento e cartão
- `reserve-button-test-cases.md` → casos de teste do botão "Reservar"
- `car-rental-test-cases.md` → casos de teste da locação de carro
- `test-summary-and-bugs.md` → resumo da execução e bugs encontrados

## Competências demonstradas
- Testes manuais
- Criação de checklist
- Escrita de casos de teste
- Validação de layout
- Validação funcional
- Bug reporting
- Organização da documentação
- Atenção aos detalhes

## Observação
Este projeto foi desenvolvido como atividade prática de formação em QA, com foco em documentação, execução de testes e análise de inconsistências no fluxo de carsharing.
